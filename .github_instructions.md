# üì¶ GitHub Repository Setup Instructions

Follow these steps to push your project to GitHub.

## üöÄ Quick Start

### Step 1: Create GitHub Repository

1. Go to [GitHub](https://github.com) and login
2. Click the **"+"** icon in the top right
3. Select **"New repository"**
4. Fill in the details:
   - **Repository name**: `Student-Eye-Drowsiness-Detection-System`
   - **Description**: `AI-powered real-time drowsiness detection system using MediaPipe and Django`
   - **Visibility**: Choose Public or Private
   - **DO NOT** initialize with README, .gitignore, or license (we already have these)
5. Click **"Create repository"**

### Step 2: Initialize Git in Your Project

Open terminal/command prompt in your project directory:

```bash
# Navigate to project directory
cd "c:\Users\Hp\Desktop\College Project\DDS_for Student"

# Initialize git repository
git init

# Add all files to staging
git add .

# Create first commit
git commit -m "Initial commit: Student Eye Drowsiness Detection System"
```

### Step 3: Connect to GitHub

Replace `yourusername` with your actual GitHub username:

```bash
# Add remote repository
git remote add origin https://github.com/yourusername/Student-Eye-Drowsiness-Detection-System.git

# Verify remote
git remote -v

# Push to GitHub
git branch -M main
git push -u origin main
```

### Step 4: Verify Upload

1. Go to your GitHub repository
2. Refresh the page
3. You should see all your files uploaded

## üìä GitHub Actions (Optional)

Create `.github/workflows/django.yml` for automated testing:

```yaml
name: Django CI

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Set up Python
      uses: actions/setup-python@v2
      with:
        python-version: 3.8
    - name: Install Dependencies
      run: |
        python -m pip install --upgrade pip
        pip install -r requirements.txt
    - name: Run Tests
      run: |
        python manage.py test
```

## üîê Security Best Practices

1. **Never commit:**
   - Database files with user data
   - Secret keys or passwords
   - API keys or tokens
   - Personal information

2. **Use environment variables:**
   - Create `.env` file for secrets
   - Add `.env` to `.gitignore`
   - Use `python-decouple` for configuration

3. **Update Django SECRET_KEY:**
   ```python
   # In settings.py
   from decouple import config
   SECRET_KEY = config('SECRET_KEY')
   ```
